Applying different a different color is very simple. Just add `theme_color: hotpink` to the page‚Äôs frontmatter.

This page‚Äôs relevant frontmatter:

```yaml
---
theme_color: hotpink
---
```

Or the whole site can have the same color declared in the site‚Äôs `_config.yml`.

This site‚Äôs relevant config:

```yaml

  theme_color: yellowgreen
```

But for this to work, the colors have to be defined beforehand. This theme has a couple defined, and they can be extended pretty easily.

This page‚Äôs `hotpink.scss`:

```scss
---
---
@use "black-white";

:root {
  --bg-color: hsl(330 100% 71%);
  --pre-bg-color: hsl(330 100% 66%);
}
```

For a style with saturated colors, only `--bg-color` and `--pre-bg-color` have to be changed.

For the `--pre-bg-color` color I mostly choose a 5 percentage points lower or higher luminance than the `--bg-color`. That‚Äôs the second percentage of this `HSL` color. [Mozilla‚Äôs color picker](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_Colors/Color_picker_tool) to the rescue.

## Create Your Own

To have a defined color in frontmatter's `theme_color` or `_config.yml`‚Äôs `theme_color` be applied, a SCSS file has to exist in `assets/css`. The files in `_sass` don‚Äôt need to be changed. Both are located in the theme itself, and not the demo you are using now.

### Named Colors

For named colors like ‚Äúorange‚Äù, the file has to be named `orange.scss`. Create a directory called `assets` in your site‚Äôs root directory, and a subdirectory called `css`. In there create a file called `orange.scss`, and put the corresponding HSL colors in there:

```css
---
---
@use "black-white";

:root {
  --bg-color: hsl(39 100% 50%);
  --pre-bg-color: hsl(39 100% 45%);
}
```

### Hex Colors

For hex codes like `#c34cff`, the file has to be named `c34cff.scss` without the preceding hash. That‚Äôs because Jekyll excludes file starting with a hash from its build process. And because file names can be case-sensitive, the code should always be lower-case.

But in frontmatter, the color has to be declared with the hash and in quotes, like `theme_color: "#c34cff"`.

Other ways of expressing HMTL colors are not supported, because the file naming and application in a page‚Äôs meta tags becomes cumbersome.
